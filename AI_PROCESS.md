## 主要开发阶段

### 1. 基础游戏与局域网访问

先搭出了一个浏览器里的炉石风小游戏，核心目标是：

- 电脑和手机都能打开
- 同一局域网内设备都能访问
- 不依赖前端框架

这一阶段主要完成了：

- Node 静态服务与页面路由
- `0.0.0.0` 监听
- `/api/meta` 输出局域网地址
- `/api/healthz` 健康检查

### 2. 主游戏页与 `/agents` 页面拆分

一开始 agent 信息放在游戏页里，后面拆成独立的 `/agents` 页面。

原因是：

- 游戏页应该以玩法为主
- agent 日志、原始交互、状态面板内容比较长
- 侧边栏不够放完整记录

后续 `/agents` 页逐步补上了：

- manager / agents 当前状态
- 交互时间线
- 更细的任务记录
- 原始日志区

### 3. 卡牌编辑器从“数值面板”迭代成“结构化编辑器”

编辑器最早只能改基础数值，后面不断暴露更细的字段。

这一阶段遇到的主要问题：

- 上面的结构化字段和下面的描述文本会漂移
- 保存时会把刚改过的机制字段覆盖回去
- 一些目标逻辑只能去改 JSON
- 自定义卡牌不一定能完整回流到运行时

修正方向是：

- 让 `editorModel` 成为结构化真源
- 自动描述从结构化字段生成
- 保存时避免 destructive re-render
- 把常用效果和条件效果做成显式字段

### 4. 单人 / 局域网 PvP 的目标系统统一

后面重点修了一轮“编辑器里能改，但实战里不一定按这个结算”的问题。

当时发现的典型问题：

- PvP 服务端目标校验和编辑器目标能力不一致
- `由玩家决定` 的法术不能真正自由选目标
- `相同目标` 这类条件后续效果没有跟随第一次选中的对象
- 法术目标高亮不完整，玩家不容易看出哪些能点

最后统一成了：

- `由玩家决定` 可以选任意英雄或任意随从，包含友方与敌方
- `相同目标` 会跟随主效果第一次选到的目标
- 前端所有可选目标都会高亮
- PvP 服务端也按同样的目标规则校验和结算

### 5. 局域网 PvP 基础链路修复

双人模式加上后，又集中修了一轮联机问题。

当时确认过的问题包括：

- 按玩家过滤状态时仍有隐藏信息泄漏风险
- 房间状态消息个性化不正确
- 断线、退房、解散房间流程不稳
- 某些大厅恢复链路会访问不存在的 UI 元素

处理方向：

- 让房间状态和对局状态按玩家各自过滤
- 修掉 lobby 恢复路径里的未定义 UI 访问
- 补上稳定 `playerId / clientId` 链路
- 断线后给房间和对局一段恢复窗口

### 6. 开场动画与抽牌节奏调整

为了更接近你指定的单关规则，开场和抽牌规则也重做过一轮。

这一阶段实现了：

- 接近全屏的先后手提示动画
- 先手开局 3 张，自己第一回合开始再抽 1 张
- 后手开局 4 张，自己第一回合开始再抽 1 张
- 之后每回合开始都抽 1 张
- 打出去的牌离开手牌，没打的继续留在手里

### 7. 关键词体系接入

后面增加了共享关键词模块，并同时接到单人和 PvP。

实现过的关键词有：

- `嘲讽`
- `剧毒`
- `复生`
- `圣盾`
- `吸血`
- `风怒`

这一轮的重要点：

- 关键词逻辑集中到共享模块，不再散落复制
- 运行时随从状态开始显式跟踪 `圣盾 / 复生可用 / 已攻击次数 / 每回合攻击上限`
- 编辑器和召唤物也能配置关键词

### 8. 复生细节修复

在关键词体系接入后，又遇到一个很关键的规则问题：

- `圣盾 + 复生` 的随从复生后，圣盾被错误清掉了

修正后的规则是：

- 复生后保留其他关键词效果
- 保留正常攻击力
- 只把生命值改成 `1`
- `复生` 本身按一次性消耗

### 9. 单人测试版加入

为了让一个人也能反复验证目标、攻击、回合和关键词效果，后来加了 `本地测试版`。

测试版规则是：

- 对手名为 `测试陪练`
- 每个敌方回合获得 `5` 点护甲
- 每个敌方回合召唤 `1` 个 `2/2` 随从

这样做的原因是：

- 不需要第二台设备
- 不需要总是走完整局 PvP 建房流程
- 更适合快速验法术目标、攻击次数、关键词和回合节奏

### 10. 单人进度保存与 PvP 断线恢复

后来又补了刷新恢复能力，因为单人和联机都存在“刷新后全丢”的体验问题。

最后形成的方案：

- 单人 `Boss / 本地测试版` 进度保存在浏览器本地
- URL 使用不同模式参数，如 `?mode=solo&scenario=test`
- PvP 使用稳定浏览器本地身份
- 服务端为断线玩家保留短时间恢复窗口
- 同一设备刷新或重新打开后会尝试回到原房间或当前对局

## 本轮重点修过的问题

### 编辑器问题

- 描述文本和结构化字段不同步
- 改 `条件奖励值` 时保存后被旧值覆盖
- 条件目标选项不够
- 机制修改一部分只能靠手写 JSON

### 目标系统问题

- `背水一击` 等玩家自选目标法术不能真正选任意目标
- `相同目标` 没有跟随主效果目标
- 高亮提示不完整

### PvP 问题

- 对局状态可能泄漏不该看到的信息
- 法术目标在 PvP 里结算不一致
- 房间状态个性化错误
- 断线、退房、恢复过程不稳定

### 规则问题

- `圣盾 + 复生` 复生后丢失圣盾
- 关键词显示顺序不一致
- `复生` 显示顺序后来固定放到最后
